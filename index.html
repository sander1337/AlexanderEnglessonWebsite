Overview

I built a fully custom particle and VFX editor in C++ inside our team's engine (TGE), designed for real production use alongside artists and designers. The system supports seven spawn shapes, nine over-lifetime curves (size, velocity XYZ, colour RGBA, rotation), a sub-emitter architecture, a configurable burst system, PhysX world collision with decal spawning, and a warmup system for pre-simulating effects before the scene starts. Every property round-trips through a JSON serialisation layer, and the editor renders live per-shape wireframe debug overlays directly into the viewport so emitter bounds are always visible while tuning.

My Role & Responsibilities

I designed and built the full system solo — the data architecture, the ImGui editor UI, the JSON serialisation layer, the runtime particle update loop, the PhysX collision integration, and the shader integration. I also wrote the DrawShapeDebug static function that renders shape-accurate wireframes into the editor viewport, and a custom Rodrigues rotation implementation to orient all spawn shapes to an arbitrary axis without gimbal lock.

Technical Highlights

The data layer uses a CopyOnWriteWrapper<ParticleEmitterData> — the editor holds a read-only view and only allocates a mutable copy the moment a field is changed, making the system safe for undo and sub-emitter nesting without defensive copying. Spawn shapes are oriented to any arbitrary axis using a custom Rodrigues rotation that avoids gimbal lock — the same helper runs in both the runtime emitter and the debug wireframe draw, keeping visual and actual spawn positions in perfect sync. The colour over-lifetime editor renders a live 50-segment gradient bar into the ImGui draw list by evaluating all four channel curves simultaneously, giving artists instant colour feedback before any particle is simulated.

Development Breakdown

Step 01 — Data Architecture
Sub-Emitter Nesting with Copy-on-Write

Each emitter is a CopyOnWriteWrapper<ParticleEmitterData> that owns a flat list of child sub-emitters. The editor holds a read-only view and only calls .Edit() at the exact moment a field changes. Sub-emitters are added, named, and removed at runtime from collapsing headers in the editor panel. The whole structure serialises symmetrically — every field written by WriteToJson is read back by LoadFromJson with presence checks so older assets load cleanly without crashing.

ParticleEmitterData — JSON round-trip C++ 
// LoadFromJson — presence-checked for back-compat
if (someJson.contains("subEmitters"))
{
  for (auto& subJson : someJson["subEmitters"])
  {
    CopyOnWriteWrapper<ParticleEmitterData> subWrapper =
      CopyOnWriteWrapper<ParticleEmitterData>::Create();
    LoadFromJson(subWrapper, subJson);
    d.subEmitters.push_back(subWrapper.Get());
  }
}

// WriteToJson — mirrors every field symmetrically
nlohmann::json subEmittersJson = nlohmann::json::array();
for (auto& sub : d.subEmitters)
{
  nlohmann::json subJson;
  CopyOnWriteWrapper<ParticleEmitterData> subWrapper =
    CopyOnWriteWrapper<ParticleEmitterData>::Create();
  subWrapper.Edit() = sub;
  WriteToJson(subWrapper, subJson);
  subEmittersJson.push_back(subJson);
}
someJson["subEmitters"] = subEmittersJson;
Step 02 — Over-Lifetime Curves
Nine Editable Curves with Live Colour Preview

Size, velocity (X/Y/Z), colour (R/G/B/A), and rotation are all curve-driven. The colour section evaluates all four channel curves simultaneously and renders a live 50-segment gradient bar directly into the ImGui draw list — so artists see the full colour transition instantly before any particle is spawned. Each velocity channel has its own sub-tab (X / Y / Z) with independent curve editing. All curve data flows through to the pixel and vertex shaders at runtime, keeping editor preview and in-game output in sync.

Colour gradient preview bar C++ 
// Evaluate all 4 channel curves at normalised time t
auto EvaluateColorAt = [&](float t) -> ImVec4
{
  auto eval = [&](const std::vector<CurveKey>& curve)
  {
    if (curve.empty()) return 1.f;
    for (size_t i = 1; i < curve.size(); ++i)
    {
      if (t <= curve[i].time)
      {
        float localT = (t - curve[i-1].time)
                     / (curve[i].time - curve[i-1].time);
        return curve[i-1].value * (1.f - localT)
             + curve[i].value * localT;
      }
    }
    return curve.back().value;
  };
  return ImVec4(eval(data.colorOverLifetimeR),
                eval(data.colorOverLifetimeG),
                eval(data.colorOverLifetimeB),
                eval(data.colorOverLifetimeA));
};

// Draw 50-segment bar directly into the ImGui drawlist
ImVec2 barPos = ImGui::GetCursorScreenPos();
for (int i = 0; i < 50; ++i)
{
  float t0 = float(i) / 50;
  ImVec4 col = EvaluateColorAt(t0);
  ImGui::GetWindowDrawList()->AddRectFilled(
    ImVec2(barPos.x + t0 * barSize.x, barPos.y),
    ImVec2(barPos.x + (t0 + 0.02f) * barSize.x,
           barPos.y + barSize.y),
    IM_COL32(col.x*255, col.y*255,
             col.z*255, col.w*255));
}
Step 03 — Spawn Shapes
Seven Shapes Oriented via Rodrigues Rotation

The system supports Point, Sphere, Hollow Sphere, Cone, Box, Circle, and Edge. Every shape except Point is oriented to an arbitrary spawnDirection vector using a custom Rodrigues rotation — a compact formula that rotates any local-space vector onto a target axis in a single pass, with explicit edge-case handling for parallel and anti-parallel inputs. The same RotateOntoAxis lambda runs inside both CalculateParticleSpawn and DrawShapeDebug, so the debug wireframe always matches the actual spawn volume exactly.

CalculateParticleSpawn — Rodrigues rotation C++ 
// Rodrigues rotation: rotates local-space vector v
// onto target axis in one pass — no gimbal lock
auto RotateOntoAxis = [](Vector3f v, Vector3f axis) -> Vector3f
{
  const Vector3f up = { 0.f, 1.f, 0.f };
  float dot = axis.x*up.x + axis.y*up.y + axis.z*up.z;

  // Edge cases: already aligned or exactly opposite
  if (std::abs(dot - 1.f) < 1e-5f) return v;
  if (std::abs(dot + 1.f) < 1e-5f)
    return { -v.x, -v.y, -v.z };

  // Rotation axis k = normalize(up × axis)
  Vector3f k = {
    up.y*axis.z - up.z*axis.y,
    up.z*axis.x - up.x*axis.z,
    up.x*axis.y - up.y*axis.x
  };
  float kLen = std::sqrt(k.x*k.x + k.y*k.y + k.z*k.z);
  k = { k.x/kLen, k.y/kLen, k.z/kLen };

  float a = std::acos(std::clamp(dot, -1.f, 1.f));
  float c = std::cos(a), s = std::sin(a), t = 1.f - c;

  return {
    (t*k.x*k.x+c)*v.x    + (t*k.x*k.y-s*k.z)*v.y + (t*k.x*k.z+s*k.y)*v.z,
    (t*k.x*k.y+s*k.z)*v.x + (t*k.y*k.y+c)*v.y    + (t*k.y*k.z-s*k.x)*v.z,
    (t*k.x*k.z-s*k.y)*v.x + (t*k.y*k.z+s*k.x)*v.y + (t*k.z*k.z+c)*v.z
  };
};
Step 04 — Shape Debug Overlay
Live Per-Shape Wireframes in the Editor Viewport

DrawShapeDebug is a static function that draws shape-accurate wireframes directly into the engine's debug draw layer — a full ring for Sphere, a cone silhouette with base ring and side lines for Cone, a full 12-edge box for Box, and so on. It runs every frame inside ObjectDefinitionDocument::Update after the render pipeline so the lines composite on top of the live particle preview. Because it uses the same RotateOntoAxis helper as the runtime emitter, the wireframe always matches the actual spawn volume — editing the cone angle live moves both the particles and the debug outline together.

DrawShapeDebug — Cone & Box cases C++ 
case Shape::Cone:
{
  float h = ed.coneHeight;
  float r = h * tanf(ed.coneAngle * PI / 180.f);
  Vector3f apex = center + axis * h;

  for (int i = 0; i < segments; i++)
  {
    float a0 = (float)i / segments * 2.f * PI;
    float a1 = (float)(i+1) / segments * 2.f * PI;

    // Base ring — oriented onto spawnDirection
    Vector3f p0 = center +
      RotateOntoAxis({ cosf(a0)*r, 0.f, sinf(a0)*r }, axis);
    Vector3f p1 = center +
      RotateOntoAxis({ cosf(a1)*r, 0.f, sinf(a1)*r }, axis);

    AddLine(p0, p1);   // base ring segment
    AddLine(apex, p0); // side line to apex
  }
  break;
}
case Shape::Box:
{
  float hx = ed.boxSize.x*0.5f,
        hy = ed.boxSize.y*0.5f,
        hz = ed.boxSize.z*0.5f;
  // Bottom face + top face + 4 vertical edges
  AddLine({center.x-hx,center.y-hy,center.z-hz},
          {center.x+hx,center.y-hy,center.z-hz});
  // ... (12 edges total)
  break;
}
Step 05 — Burst System
Timed Bursts with Cycles and Intervals

Each emitter supports any number of bursts, each with a trigger time, particle count, cycle count (−1 for infinite repeat), and repeat interval. Burst state is tracked per-burst in a BurstState struct that persists across frames — when a burst fires, it increments cyclesFired and schedules the next fire time by adding the interval. A cycle count of −1 keeps it firing indefinitely. Bursts can be added and removed live in the editor panel. This made it straightforward to author a boss ability that emits a large burst at a specific moment and then pulses repeatedly for the rest of the attack.

ParticleEmitter.cpp — burst tick C++ 
// Per-burst state tracked across frames
struct BurstState
{
  int   cyclesFired  = 0;
  float nextFireTime = -1.f; // -1 = not yet armed
};

// Evaluated every Update() per emitter
for (size_t i = 0; i < emitter.data.bursts.size(); i++)
{
  auto& state = emitter.burstStates[i];
  if (state.nextFireTime < 0.f) continue;

  if (emitter.emitterElapsed >= state.nextFireTime)
  {
    for (int b = 0; b < emitter.data.bursts[i].count; b++)
      SpawnParticle(emitter);

    state.cyclesFired++;
    bool infinite = emitter.data.bursts[i].cycles < 0;
    if (infinite || state.cyclesFired < emitter.data.bursts[i].cycles)
      state.nextFireTime += emitter.data.bursts[i].interval;
    else
      state.nextFireTime = -1.f; // disarm
  }
}
Step 06 — PhysX Collision & Decal Spawning
Per-Particle Raycasts with World Hit Response

When collideWithWorld is enabled, each particle's previous position is stored every frame. In Update, a PhysX raycast fires from the previous position toward the current one along the exact travel delta — so fast-moving particles never tunnel through geometry. On a confirmed hit, two independent behaviours trigger: the particle can be force-expired (dieOnCollision), and a TGO decal can be spawned at the hit world position with a configurable scale. The decal TGO is assigned by drag-dropping an asset directly onto the editor field.

ParticleEmitter.cpp — per-particle PhysX raycast C++ 
if (emitter.data.collideWithWorld && globalCurrentPxScene)
{
  // Ray from previous position to current position
  physx::PxVec3 origin = { prevPos.x, prevPos.y, prevPos.z };
  physx::PxVec3 delta  = { pos.x - prevPos.x,
                            pos.y - prevPos.y,
                            pos.z - prevPos.z };
  float dist = sqrtf(delta.x*delta.x +
                     delta.y*delta.y +
                     delta.z*delta.z);

  if (dist > 0.001f)
  {
    physx::PxVec3 dir = { delta.x/dist,
                          delta.y/dist,
                          delta.z/dist };
    physx::PxRaycastBuffer hit;

    if (globalCurrentPxScene->raycast(origin, dir, dist, hit))
    {
      // Spawn decal TGO at world hit point
      if (emitter.data.spawnDecalOnCollision &&
          !emitter.data.collisionTgoName.IsEmpty())
      {
        Vector3f hitPos = { hit.block.position.x,
                            hit.block.position.y,
                            hit.block.position.z };
        Matrix4x4f spawnTM =
          Matrix4x4f::CreateFromTranslation(hitPos);
        spawnTM.SetScale(emitter.data.collisionTgoScale);
        GameWorld::GetInstance().LoadAssetToScene(
          emitter.data.collisionTgoName, spawnTM, true);
      }
      if (emitter.data.dieOnCollision)
        vfx->ForceExpire();
    }
  }
}
